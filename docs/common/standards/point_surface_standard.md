# Point Surface Standard

This filename is retained for taxonomy parity with the reference standards set. In this repo it governs the analogous direct one-shot public surfaces.

## Purpose

This standard defines the expectations for direct convenience surfaces that execute a representative operation without requiring a long-lived prepared runtime.

## Rules

- direct surfaces should be easy to discover and call
- they should state when they are convenience wrappers rather than production optimized paths
- docs should point users to the prepared or optimized path when repeated execution matters
- direct surfaces should still emit honest diagnostics and metadata where supported
