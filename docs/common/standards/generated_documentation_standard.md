# Generated Documentation Standard

## Purpose

This standard defines how generated or partially generated docs should be handled in the common track.

## Rules

- generated documents must say they are generated or machine-refreshed
- if a generation command exists, document it in the file or neighboring governance docs
- do not hand-edit generated inventories casually without also updating the source logic
- manually maintained docs should not pretend to be generated

## Shared-Layer Interpretation

Current common-track docs are mostly hand-maintained, but JSON registries and inventory-style reports may still be generation candidates and should declare their ownership clearly.
