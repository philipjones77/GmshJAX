# Report Standard

## Purpose

This document defines what belongs in `docs/common/reports/`.

## Placement Rule

Put a document in `docs/common/reports/` when the main question is:

- what currently exists
- what shared surface is exposed
- what inventory, matrix, or registry is available

## Writing Rule

Reports should:

- describe the current repo state
- use factual inventory language
- point to the governing standard when policy is relevant
- avoid turning into TODO lists or roadmaps

Reports should not:

- own policy
- duplicate status tracking
- describe speculative future behavior as if it already exists
