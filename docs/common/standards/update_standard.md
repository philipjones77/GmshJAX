# Update Standard

## Purpose

This standard defines what must be updated when the shared surface changes materially.

## Rules

- update the owning standard or spec first if the contract changed
- update `src/common/` implementation and any affected adapters
- refresh shared reports or registries when their inventories changed
- update tests, benchmarks, examples, and notebooks that demonstrate the changed surface
- update common status docs when the change closes or opens a tracked gap
