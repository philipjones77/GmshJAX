# Theory Notation Standard

## Purpose

This standard defines the split between notation, theory, and implementation language in the shared track.

## Rules

- shared terminology belongs in `docs/common/notation/`
- conceptual explanations of mode vocabulary, bridge methodology, and shared abstractions belong in `docs/common/theory/`
- implementation walkthroughs belong in `docs/common/implementation/`
- track names, mode names, runtime names, and bridge terms should be used consistently across docs
