# Common Repo Standards Verification

The shared track should be checked against these standards:

- common-facing APIs use stable naming and clear ownership
- import boundaries stay narrow and lazy where intended
- diagnostics and report payloads remain JSON-safe
- mode vocabulary stays aligned across common, Topo, SMPL, and NumPy runtime paths
- shared docs are placed in `docs/common/` rather than duplicated ad hoc elsewhere

This report is a lightweight checkpoint document and should be refreshed when the common surface changes materially.
