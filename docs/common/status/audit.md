# Common Track Audit

Current strengths:

- shared backend registry exists
- shared IO and diagnostics helpers exist
- shared movement transform exists
- NumPy runtime exists for non-AD mesh workflows
- Topo and SMPL adapters exist through the common layer

Current gaps to keep watching:

- shared docs need regular refresh as adapters evolve
- inventories and reports should be kept in sync with exported surfaces
- common examples and benchmarks should continue to distinguish shared behavior from backend cost
